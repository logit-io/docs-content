Metricbeat is a lightweight shipper that helps you monitor your Redis 
servers by collecting metrics running on the Redis server. 
Configure Metricbeat using the pre-defined examples below to collect and 
ship Redis service metrics and statistics to Logstash or Elasticsearch.

<Steps>
  ### Install Integration
  <InstallIntegration/>
  ### Install Metricbeat

  <InstallBeat beatname="metricbeat" />

  ### Copy Configuration File

  <MetricbeatOutput />

  ### Enable module

  <EnableMetricbeatModule module="redis" />

  ### Configure Module

  <ConfigureMetricbeatModule module="redis">
    ```yml copy showLineNumbers
    - module: redis
      metricsets:
      # - info
      # - keyspace
      period: 10s
    ```
  </ConfigureMetricbeatModule>

  ### Start Metricbeat

  [Start or restart metricbeat](https://www.elastic.co/guide/en/beats/metricbeat/master/metricbeat-starting.html) 
  to apply the configuration changes.

  ### Check Logit.io for your logs

  <LaunchStack utmMedium="logs" utmCampaign="redis_metricbeat" source="metrics_redis" />

  ### How to diagnose no data in Stack

  <DiagnoseNoData />

  ### Compatability

  The Redis metricsets were tested with Redis `3.2.12`, `4.0.11` and `5.0-rc4`, 
  and are expected to work with all versions >= 3.0.

</Steps>
