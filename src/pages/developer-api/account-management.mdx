# Account Management via API

Manage account information, users, teams, settings, and invoices using the Logit.io Developer API. These endpoints support multi-account workflows, billing automation, and access control.

Use your Dashboard API key from the [Profile](https://dashboard.logit.io/profile) page in the `x-api-key` header. See [Authentication](/developer-api/authentication) for setup and security.

## List Accounts

Get a list of accounts accessible to the authenticated user. You can filter by enabled and paying status:

<Tabs items={["Linux / macOS", "Windows"]}>
<Tab>

```bash copy /@dashboardApiKey/
# Get all accounts
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/accounts

# Filter by enabled status (true, false, or omit for no filter)
curl -H "x-api-key: @dashboardApiKey" "https://dashboard.logit.io/api/accounts?Enabled=true"

# Filter by paying status
curl -H "x-api-key: @dashboardApiKey" "https://dashboard.logit.io/api/accounts?Paying=true"
```
</Tab>
<Tab>

```cmd copy /@dashboardApiKey/
# Get all accounts
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/accounts

# Filter by enabled status (true, false, or omit for no filter)
curl -H "x-api-key: @dashboardApiKey" "https://dashboard.logit.io/api/accounts?Enabled=true"

# Filter by paying status
curl -H "x-api-key: @dashboardApiKey" "https://dashboard.logit.io/api/accounts?Paying=true"
```
</Tab>
</Tabs>

The response includes account IDs, display names, and status for each account.

## Get Account Stacks

Retrieve all stacks (Logs, Metrics, APM) grouped by subscription for an account:

<Tabs items={["Linux / macOS", "Windows"]}>
<Tab>

```bash copy /@dashboardApiKey/
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/account/ACCOUNT_ID/stacks
```
</Tab>
<Tab>

```cmd copy /@dashboardApiKey/
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/account/ACCOUNT_ID/stacks
```
</Tab>
</Tabs>

Replace `ACCOUNT_ID` with the account GUID. Use this to discover stacks before calling stack-specific endpoints.
