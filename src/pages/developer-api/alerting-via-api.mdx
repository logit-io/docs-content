# Alerting via API

Manage alerting for your Logs and Infrastructure Metrics stacks via the Logit.io Developer API. This guide covers enabling alerting, listing and creating alert rules, and configuring AlertManager (Metrics).

For an overview of alerting concepts, see [Log Management Alerting](/log-management/alerting/overview) and [Infrastructure Metrics Alerting](/infrastructure-metrics/alerting/overview).

## Enable or Disable Alerting

For a stack with alerting available:

<Tabs items={["Linux / macOS", "Windows"]}>
<Tab>

```bash copy /@dashboardApiKey/
# Enable alerting (provisions alerting server)
curl -X POST -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/enable

# Disable alerting (decommissions alerting server)
curl -X POST -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/disable

# Get alerting status
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/status
```
</Tab>
<Tab>


### Create an Alert Rule

<Tabs items={["Linux / macOS", "Windows"]}>
<Tab>

```bash copy /@dashboardApiKey/
curl -X POST -H "x-api-key: @dashboardApiKey" -H "Content-Type: application/json" \
  -d '{"newFileName": "my-alert.yaml", "templateFileName": "frequency"}' \
  https://dashboard.logit.io/api/stacks/STACK_ID/alerting/rules
```
</Tab>
<Tab>

```cmd copy /@dashboardApiKey/
curl -X POST -H "x-api-key: @dashboardApiKey" -H "Content-Type: application/json" ^
  -d "{\"newFileName\": \"my-alert.yaml\", \"templateFileName\": \"frequency\"}" ^
  https://dashboard.logit.io/api/stacks/STACK_ID/alerting/rules
```
</Tab>
</Tabs>

Use `templateFileName` to start from an existing template, or omit it. The response includes the new rule ID.

## Infrastructure Metrics (AlertManager and Metrics Rules)

For Metrics stacks, you can manage AlertManager configuration and metrics alert rules (Prometheus-style YAML).

### AlertManager Configuration

<Tabs items={["Linux / macOS", "Windows"]}>
<Tab>

```bash copy /@dashboardApiKey/
# Get AlertManager config
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/alert-manager

# Save AlertManager config (PUT with configYaml and optional templateContent)
curl -X PUT -H "x-api-key: @dashboardApiKey" -H "Content-Type: application/json" \
  -d '{"configYaml": "global:\n  resolve_timeout: 5m\nroute:\n  receiver: default", "templateContent": null}' \
  https://dashboard.logit.io/api/stacks/STACK_ID/alerting/alert-manager

# Test AlertManager config (POST), then poll test-results/TEST_RUN_ID
# Apply after test succeeds (PUT /alert-manager/apply/TEST_RUN_ID)
```
</Tab>
<Tab>


### Metrics Alert Rules

<Tabs items={["Linux / macOS", "Windows"]}>
<Tab>

```bash copy /@dashboardApiKey/
# List metrics rules
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/metrics-rules

# Get or update a rule by file name (e.g. alerts.yml)
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/metrics-rules/alerts.yml
curl -X PUT -H "x-api-key: @dashboardApiKey" -H "Content-Type: application/json" \
  -d '{"rulesYaml": "groups:\n- name: example\n  rules:\n  - alert: HighErrorRate"}' \
  https://dashboard.logit.io/api/stacks/STACK_ID/alerting/metrics-rules/alerts.yml
```
</Tab>
<Tab>

```cmd copy /@dashboardApiKey/
# List metrics rules
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/metrics-rules

# Get or update a rule by file name (e.g. alerts.yml)
curl -H "x-api-key: @dashboardApiKey" https://dashboard.logit.io/api/stacks/STACK_ID/alerting/metrics-rules/alerts.yml
curl -X PUT -H "x-api-key: @dashboardApiKey" -H "Content-Type: application/json" ^
  -d "{\"rulesYaml\": \"groups:\n- name: example\n  rules:\n  - alert: HighErrorRate\"}" ^
  https://dashboard.logit.io/api/stacks/STACK_ID/alerting/metrics-rules/alerts.yml
```
</Tab>
</Tabs>

Workflow for Metrics rules: save (PUT) → test (POST) → poll test-results → apply (PUT /apply/TEST_RUN_ID) when status is Success.
